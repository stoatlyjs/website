stoatly.js

string‑based command framework for Stoat · built on stoat.js
v0.1.0 CI passing MIT License

A string‑based command framework for Stoat (the open‑source chat platform formerly known as Revolt), built on the official stoat.js client.

Instead of writing raw event handlers, write commands as strings containing $functions: 

import { StoatlyClient } from "stoatly.js";

const client = new StoatlyClient({ prefix: "!" });

client.command({
  name: "ping",
  code: "Pong! $ping ms",
});

client.command({
  name: "say",
  aliases: ["echo"],
  code: "$onlyIf[$argsCount>0;You need to give me something to say!]$sendMessage[$args]",
});

client.command({
  name: "score",
  code: `
    $if[$args[0]==add;
      $addVar[score;1;$authorID]You're now at $getVar[score;0;$authorID] points!;
      Your score is $getVar[score;0;$authorID]
    ]
  `,
});

client.login("YOUR_BOT_TOKEN");

⚙️ Requirements

  • • Node.js v22.15.0+ (or Deno v2.2+)
  • • stoatly.js is an ES module — use import, not require().
  • • If your project is CommonJS, either switch to "type": "module" or load with dynamic import("stoatly.js").

📦 Install

npm install stoatly.js

stoat.js ships as a dependency (pinned to ^7.3.6). No need to install it separately.

For raw access to the underlying Stoat client: client.client gives you the stoat.js Client instance.

🧩 How the DSL works

  • • $functionName[arg1;arg2;...] — calls a function with semicolon‑separated args.
  • • $functionName (no brackets) calls with zero args (e.g. $ping).
  • • Arguments can contain nested $functions.
  • • $$ produces a literal $.
  • • Plain [ / ] are fine — the parser tracks bracket depth. But ; always acts as a separator, so avoid stray semicolons inside arguments (use $getVar‑style separate args).
  • • Lazy functions ($if, $onlyIf, $repeat) get raw, unevaluated arguments so they can control evaluation (e.g. avoid side effects in the “else” branch).

🔧 Built‑in functions

Context / reading

FunctionDescription
$message[index?]Full message content, or a specific word by index
$args[index?]All args joined, or one arg by index
$argsCountNumber of args passed to the command
$mention[index?]Formats a mentioned user's ID as <@id>
$authorIDID of the message author
$usernameUsername of the message author
$channelIDCurrent channel ID
$serverIDCurrent server ID
$prefixThe prefix that triggered this command
$pingClient latency in ms

Actions

FunctionDescription
$sendMessage[content;channelID?]Sends a message
$reply[content]Replies to the triggering message
$deleteMessage[delayMs?]Deletes the last sent (or triggering) message
$editMessage[content]Edits the last message sent via $sendMessage
$addReaction[emoji]Reacts to the last sent (or triggering) message
$dm[userID;content]Sends a direct message to a user
$wait[ms]Pauses execution

Logic

FunctionDescription
$if[condition;then;else?]Branches; supports ==,!=,>,<,>=,<=,&&,||
$onlyIf[condition;errorMessage?]Stops the command if the condition is false
$repeat[count;code]Runs code up to count times (max 1000)
$loopIndexCurrent index inside $repeat (0‑based)

Variables

Persisted to a JSON file by default (swap in your own database).

FunctionDescription
$setVar[name;value;scope?]Stores a value (scope defaults to the server ID)
$getVar[name;fallback?;scope?]Reads a value
$addVar[name;amount;scope?] / $subVarIncrements/decrements a numeric value
$deleteVar[name;scope?]Deletes a value
$hasVar[name;scope?]"true" / "false"

Utility

FunctionDescription
$math[expression]Safe arithmetic: + - * / % ^ ()
$random[min;max]Random integer, inclusive
$randomText[a;b;c;...]Picks one argument at random
$upperCase[text] / $lowerCaseCase conversion
$comment[anything]Evaluates to nothing (for notes in your code)
$newlineInserts \n

🧪 Custom functions

client.addFunction({
  name: "double",
  execute: (args) => String(Number(args[0]) * 2),
});

For lazy functions (like $if), set lazy: true and read execute(rawArgNodes, ctx, evalNodes).

⏫ Escaping to raw stoat.js

You can still listen to raw events directly:

client.on("messageCreate", (message) => {
  // full access to the underlying stoat.js Message object
});

client.client gives you the underlying stoat.js Client instance for lower‑level access. The objects (users, channels, messages) are reactive — great for building a UI on top of a bot.

stoat.js re‑exports raw Stoat API types under API (import { API } from "stoat.js").

📁 Project layout

stoatly.js/
  index.js                 - public exports
  src/
    StoatlyClient.js          - client, command registry, event wiring
    parser.js                - turns "code" strings into an AST
    Interpreter.js           - walks the AST, calls functions
    expressions.js           - safe $math / $if condition evaluators
    Database.js              - simple JSON-file variable store
    functions/
      context.js              - $message, $mention, $authorID, ...
      actions.js              - $sendMessage, $reply, $wait, ...
      logic.js                - $if, $onlyIf, $repeat
      variables.js            - $setVar, $getVar, ...
      utility.js              - $math, $random, ...
